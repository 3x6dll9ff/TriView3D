# Лог ключевых решений

## 1. Отказ от intensity-based layering
**Дата:** 2026-03-23
**Было:** 2D image → intensity → height map → 3D voxel
**Стало:** 2D slices → learned autoencoder → 3D shape
**Почему:** Intensity layering не имеет физического обоснования. Яркость пикселя ≠ высота клетки. На защите это легко атакуемый аргумент. Learned reconstruction (как в SHAPR) — стандартный и научно обоснованный подход.

## 2. Отказ от синтетических данных как основы
**Дата:** 2026-03-23
**Было:** Генерируем синтетические 3D эллипсоиды, учимся на них
**Стало:** Используем реальный SHAPR датасет (825 реальных клеток)
**Почему:** Синтетика не валидирует метод на реальных данных. На защите спросят "а на реальных клетках работает?".

## 3. Отказ от MONAI DenseNet
**Дата:** 2026-03-23
**Было:** MONAI DenseNet121 для 3D классификации
**Стало:** Custom autoencoder (2D encoder + 3D decoder) + MLP classifier
**Почему:** DenseNet = классификатор. Нам сначала нужно **реконструировать** 3D из 2D, а потом классифицировать. Autoencoder решает обе задачи: decoder восстанавливает 3D, encoder даёт latent для классификации.

## 4. Метки из имён файлов SHAPR (не KMeans)
**Дата:** 2026-03-23
**Было:** KMeans/threshold на морфометрии → бинарные метки
**Стало:** Метки напрямую из имён файлов: `discocyte` = normal, остальное = anomaly
**Почему:** В SHAPR имена файлов содержат тип клетки (discocyte, stomatocyte, echinocyte...). Score от -1.0 до 1.0 — степень деформации. Это **ground truth метки**, не нужна кластеризация.

## 5. Dual-view (2 среза) vs single-view (1 срез)
**Дата:** 2026-03-23
**Выбор:** Dual-view — основная модель, single-view — baseline
**Почему:** Основной вклад работы. Два среза (top + bottom) несут больше информации о 3D форме. SHAPR показал: из 1 среза можно предсказать 3D. Мы показываем: из 2 лучше. Количественное сравнение dual vs single = ключевой результат для диплома.

## 6. iPSC клетки — отдельная категория
**Дата:** 2026-03-23
**Решение:** Для бинарной классификации (normal/anomaly) использовать только RBC клетки (score от -1.0 до 1.0), iPSC клетки (A-E) — для дополнительного мультиклассового эксперимента.
**Почему:** RBC имеют чёткий "нормальный" тип (discocyte, score=0) и gradation аномалий. iPSC — другой тип клетки, смешивание некорректно.

## 7. Sum Projection вместо бинарного MIP
**Дата:** 2026-03-23
**Было:** MIP (max) от binary mask → одинаковые белые кляксы сверху и снизу
**Стало:** Sum Projection — сумма вокселей по оси → карта толщины с градиентом
**Почему:** Binary MIP теряет информацию о толщине (max(0,1)=1 везде). Sum Projection сохраняет сколько вокселей заполнено → discocyte даёт кольцо (края толще центра). Два ракурса (sum по Z + sum по Y) дают реально разные изображения.
